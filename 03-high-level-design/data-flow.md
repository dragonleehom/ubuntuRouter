# 数据流设计

> 版本: v1.0 | 日期: 2026-04-25

## 1. 配置数据流（主路径）

用户修改配置到系统生效的完整路径：

```
用户操作 (Web GUI / CLI)
    │
    ▼
REST API / CLI Parser
    │
    ▼
┌──────────────────────────┐
│  1. 写入 config.yaml     │  ← 单一配置源
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│  2. Schema 校验          │  ← JSON Schema / Pydantic Model
│     - 类型检查           │
│     - 语义校验           │
│     - 依赖检查           │
└───────────┬──────────────┘
            │ 校验通过
            ▼
┌──────────────────────────┐
│  3. 创建快照             │  ← /var/lib/ubunturouter/snapshots/{timestamp}.yaml
│     - 保存当前配置       │
│     - 记录变更摘要       │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│  4. 计算变更集 (Diff)    │  ← 新配置 vs 当前生效配置
│     - 新增的接口/规则    │
│     - 修改的配置         │
│     - 删除的配置         │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│  5. 生成子系统配置       │
│     ├→ netplan YAML      │
│     ├→ nftables conf     │
│     ├→ dnsmasq conf      │
│     ├→ FRR conf          │
│     └→ WireGuard conf    │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│  6. 原子 Apply           │  ← 所有配置文件先写入，再统一 reload
│     - 写入所有配置文件   │
│     - reload 各服务      │
│     - 顺序：network →    │
│       firewall → routing │
│       → dhcp → dns → vpn │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│  7. 连通性检测           │  ← 60秒定时器
│     - ping 网关          │
│     - DNS 解析测试       │
│     - API 可达测试       │
│     ─────────────────── │
│     失败 → 自动回滚     │  ← 恢复上一快照
│     成功 → 确认生效     │
└──────────────────────────┘
```

## 1.5 首次启动初始化流（特殊路径）

系统首次启动时，`/etc/ubunturouter/config.yaml` 不存在，走初始化流程：

```
系统启动 → ubunturouter-engine.service 启动
    │
    ▼
┌──────────────────────────────────────┐
│  检测 /etc/ubunturouter/config.yaml  │
│  是否存在？                           │
└──────────┬───────────────┬───────────┘
           │ 存在           │ 不存在
           ▼               ▼
     正常启动模式    ┌──────────────────────────┐
                    │  1. 网口自动探测           │
                    │     ls /sys/class/net/    │
                    │     排除 lo / docker*     │
                    │     ethtool 获取速率      │
                    │     检测链路状态          │
                    └──────────┬───────────────┘
                               │
                               ▼
                    ┌──────────────────────────┐
                    │  2. WAN/LAN 自动分配     │
                    │                          │
                    │  网口数 = 1:              │
                    │    role = wanlan         │
                    │    IP = 192.168.21.1/24  │
                    │                          │
                    │  网口数 >= 2:             │
                    │    最低速 = wan (DHCP)   │
                    │    其余 = br-lan         │
                    │    LAN = 192.168.21.1/24 │
                    └──────────┬───────────────┘
                               │
                               ▼
                    ┌──────────────────────────┐
                    │  3. 生成初始 config.yaml │
                    │     - 默认网关 .21.1     │
                    │     - DHCP .21.50-200    │
                    │     - 防火墙默认策略     │
                    │     - DNS 默认上游       │
                    └──────────┬───────────────┘
                               │
                               ▼
                    ┌──────────────────────────┐
                    │  4. Apply 初始配置       │
                    │     netplan apply →      │
                    │     nftables reload →    │
                    │     dnsmasq start →      │
                    │     unbound start →      │
                    │     API Server start     │
                    └──────────┬───────────────┘
                               │
                               ▼
                    ┌──────────────────────────┐
                    │  5. 启动初始化向导       │
                    │     监听 0.0.0.0:80      │
                    │     HTTP → HTTPS 重定向  │
                    │     等待用户完成向导     │
                    │     向导完成 → 标记已初始化│
                    └──────────────────────────┘
```

## 2. 状态查询数据流

```
Web GUI
    │
    ├── REST GET /api/v1/status/traffic ──→ 直接读取 /proc/net/dev + tc 统计
    │
    ├── REST GET /api/v1/status/connections ──→ 读取 conntrack
    │
    ├── REST GET /api/v1/status/dhcp/leases ──→ 读取 dnsmasq lease 文件
    │
    ├── REST GET /api/v1/status/wan ──→ 解析 ip addr + ping 检测
    │
    └── WebSocket /api/v1/ws/stream ──→ 推送实时流量/CPU/内存数据
```

## 3. 数据包转发流（运行时）

```
入站数据包 (WAN)
    │
    ▼
┌──────────────┐
│  NIC Driver  │
└──────┬───────┘
       │
       ▼
┌──────────────┐     ┌──────────────────────────┐
│  netfilter   │────→│ nftables 规则链           │
│  ingress     │     │ - conntrack (连接跟踪)    │
└──────┬───────┘     │ - zone policy (区域策略)  │
       │             │ - NAT (地址转换)          │
       │             └──────────────────────────┘
       ▼
┌──────────────┐
│  Routing     │────→ ip route / FRR 路由表查找
│  Decision    │
└──────┬───────┘
       │
       ▼
┌──────────────┐     ┌──────────────────────────┐
│  netfilter   │────→│ nftables forward 链       │
│  forward     │     │ - zone→zone forwarding    │
└──────┬───────┘     │ - ACL 规则               │
       │             └──────────────────────────┘
       ▼
┌──────────────┐
│  netfilter   │────→ SNAT/masquerade
│  egress      │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  NIC Driver  │────→ 出站 (LAN)
└──────────────┘
```

## 4. DNS 查询流

```
LAN 客户端 DNS 请求 (192.168.1.x → 192.168.1.1:53)
    │
    ▼
┌──────────────┐
│  Unbound     │  ← 监听 53 端口
│  (DNS缓存)   │
└──────┬───────┘
       │
       ├─ 缓存命中 → 直接返回
       │
       └─ 缓存未命中
              │
              ▼
       ┌──────────────┐
       │  AdGuard     │  ← 监听 5353 端口
       │  Home        │
       └──────┬───────┘
              │
              ├─ 黑名单命中 → 返回 0.0.0.0 (广告拦截)
              │
              └─ 允许通过
                     │
                     ▼
              ┌──────────────┐
              │  上游 DNS    │  ← 223.5.5.5 / 119.29.29.29
              └──────┬───────┘
                     │
                     ▼
              返回结果 → Unbound 缓存 → 客户端
```

## 5. DHCP 地址分配流

```
LAN 客户端 DHCP Discovery
    │
    ▼
┌──────────────┐
│  dnsmasq     │  ← 监听 67/UDP
│  (DHCP)      │
└──────┬───────┘
       │
       ├─ 检查静态绑定 (mac → ip)
       │     └─ 命中 → 分配固定 IP
       │
       └─ 动态分配池
              │
              ├─ 检查 IP 冲突 (ping probe)
              │
              └─ 分配 IP → 发送 Offer → 确认 → 写入 lease 文件
```

## 10. Dashboard 数据更新流

```
Dashboard 页面加载
    │
    ├── REST GET /api/v1/dashboard/status  ← 首次加载（一次性）
    │       返回: {系统状态 + 网络状态 + 应用状态}
    │
    └── WebSocket /api/v1/ws/dashboard?token=xxx
             │
             ├── 每 2s → {type: "traffic_update", data: {接口流量, 实时速率}}
             │    来源: /proc/net/dev + tc -s qdisc
             │
             ├── 每 10s → {type: "tunnel_status", data: {tailscale, clash, wg}}
             │    来源: 
             │      Tailscale: tailscale status --json
             │      Clash: GET http://clash:9090/proxies
             │      WireGuard: wg show wg0 dump
             │
             ├── 每 10s → {type: "app_status", data: {应用列表, 状态, 流量}}
             │    来源:
             │      Docker: docker stats --no-stream --format json
             │      VM: virsh list + virsh dominfo
             │      系统服务: systemctl list-units --type=service
             │      流量: conntrack + 各应用日志解析
             │
             ├── 每 30s → {type: "system_status", data: {cpu, mem, disk, uptime}}
             │    来源: /proc/stat, /proc/meminfo, df, uptime
             │
             └── 地图数据更新
                 来源:
                   tailscale status --json → exit node → geoip
                   clash API → proxies → geoip
                   iplookup(公网IP) → 本地位置
                       针对每个通道:
                       1. 解析出对端 IP
                       2. 查内置 GeoIP 数据库
                       3. 返回 {lat, lng, city, country}
                       4. 前端在地图上渲染节点+链路
```

**GeoIP 定位策略**：

```
方案A: 内置精简版 GeoIP 数据库（推荐，离线可用）
  - 使用 ip2location/GeoLite2 社区版
  - 约 3MB 数据库，覆盖常见 IP 段
  - 每月自动更新一次
  
方案B: 在线查询（无需本地数据库）
  - curl ipinfo.io/{ip}/json（免费版 50K/月）
  - 缓存查询结果 24h
  - 只查询节点 IP（数量少，够用）
  
方案C: 混合（推荐）
  - 首次查询用在线接口（精度高）
  - 缓存到本地 SQLite
  - 后续走本地缓存
  - 定期刷新

最终决策：本地 GeoIP DB（apt install geoip-database） +
         仅节点 IP 走在线查询作为补充
```


## 10. 流量编排数据流

```
编排流程（用户操作 → 规则生效）:

用户拖拽连线: 电视图标 → Netflix → 选择 ts-exit-us 通道
    │
    ▼
┌─────────────────────────────────────────────┐
│ 1. Web GUI 发送编排规则到 API               │
│    POST /api/v1/orchestrator/rules          │
│    { device: "电视", app: "netflix",         │
│      tunnel: "ts-exit-us", priority: 1 }    │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│ 2. Traffic Orchestrator 验证                │
│    - 设备"电视"是否存在？                    │
│    - Netflix 应用特征是否存在？              │
│    - ts-exit-us 通道是否配置？               │
│    - 规则优先级是否冲突？                    │
└──────────────────┬──────────────────────────┘
                   │ 通过
                   ▼
┌─────────────────────────────────────────────┐
│ 3. Rule Compiler 编译                       │
│    输入: 编排规则 (JSON)                    │
│    输出:                                    │
│    A) nftables set:                         │
│       set device_app_netflix {              │
│         type ether_addr . inet_service      │
│         elements = { aa:bb:cc:11:22:33 .    │
│           *.nflxvideo.net }                 │
│       }                                     │
│    B) nftables mangle chain:                │
│       ip daddr @netflix_domains              │
│       ether saddr aa:bb:cc:11:22:33         │
│       mark set 0x1001                        │
│    C) ip rule:                              │
│       from all mark 0x1001 lookup 101       │
│    D) ip route table 101:                   │
│       default dev tailscale0                │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│ 4. Apply 规则到系统                         │
│    - nft -f 加载新规则                      │
│    - ip rule add / ip route add             │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│ 5. 确认规则生效                             │
│    - nft list chain... 验证 set 正确        │
│    - ip rule list 验证 mark 规则存在         │
└─────────────────────────────────────────────┘
