# 非功能需求

> 版本: v1.0 | 日期: 2026-04-25

## 1. 性能需求

### 1.1 转发性能

| 指标 | x86_64 目标 | ARM64 目标 | 测试方法 |
|------|------------|------------|----------|
| NAT 吞吐 (1500B) | ≥ 9.4 Gbps | ≥ 2 Gbps | iperf3 单流 |
| NAT 吞吐 (64B) | ≥ 1 Gbps | ≥ 200 Mbps | iperf3 单流 |
| NAT 吞吐 (XDP加速) | ≥ 14 Gbps | N/A | XDP redirect 模式 |
| WireGuard 吞吐 | ≥ 2.5 Gbps | ≥ 500 Mbps | iperf3 over WG |
| IPSec 吞吐 | ≥ 1 Gbps | ≥ 300 Mbps | iperf3 over IPSec |
| 转发延迟增加 | < 0.5ms | < 1ms | ping 对比 |

### 1.2 并发能力

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 并发连接数 | ≥ 500,000 | conntrack 表项 |
| 并发连接建立速率 | ≥ 10,000/s | 新建连接 |
| DHCP 租约容量 | ≥ 1,000 | 单网段 |
| DNS 查询 QPS | ≥ 10,000 | Unbound 缓存命中 |

### 1.3 Web GUI 性能

| 指标 | 目标值 |
|------|--------|
| 首页加载时间 | < 1s (LAN) |
| API 响应时间 | < 200ms (P95) |
| WebSocket 推送延迟 | < 500ms |
| 并发 Web 会话 | ≥ 10 |

### 1.4 系统启动

| 指标 | 目标值 |
|------|--------|
| 冷启动到路由可用 | < 45s (SSD) |
| 冷启动到路由可用 | < 90s (eMMC/SD) |
| 冷启动到 Web GUI 可用 | < 60s (SSD) |
| 配置 Apply 时间 | < 5s |

## 2. 可靠性需求

### 2.1 配置安全

| 需求 | 说明 |
|------|------|
| 配置原子性 | 配置变更要么全部生效，要么全部不生效 |
| 自动回滚 | Apply 后 60s 内连通性检测失败，自动回滚上一版本 |
| 快照保留 | 保留最近 50 个配置快照 |
| 配置校验 | Apply 前强制校验，非法配置不允许提交 |
| 并发保护 | 同一时刻只允许一个 Apply 操作执行（文件锁） |

### 2.2 服务可用性

| 需求 | 说明 |
|------|------|
| 核心转发与 UI 解耦 | Web GUI/API 崩溃不影响路由转发 |
| 服务自动重启 | 核心服务异常退出后 systemd 自动重启 |
| 守护进程监控 | ubunturouter-engine 持续监控各服务状态 |
| 看门狗 | 可选硬件看门狗，系统无响应时自动重启 |

### 2.3 数据完整性

| 需求 | 说明 |
|------|------|
| 配置文件校验和 | 每次写入后验证文件完整性 |
| 写前备份 | 覆盖任何子系统配置前，先备份旧版本 |
| 原子写入 | 使用 write-then-rename 模式，避免半写状态 |

## 3. 安全需求

### 3.1 访问控制

| 需求 | 说明 |
|------|------|
| Web GUI 认证 | 系统 PAM 认证（用户名+密码），与系统账号一致 |
| Session 管理 | JWT Token，30 分钟无操作自动登出 |
| 登录保护 | 连续 5 次失败锁定 15 分钟 |
| 会话跟踪 | 显示当前活跃会话，支持强制登出 |
| 仅 LAN 可达 | API Server 默认仅监听 LAN 接口 |
| HTTPS | Web GUI 强制 HTTPS（自签证书，安装时生成） |
| API 鉴权 | 与 Web 共用 JWT，Bearer Token |
| 双因子认证 | TOTP 可选（预留） |
| 角色权限 | admin / viewer / operator（预留 Phase 2） |

### 3.2 系统安全

| 需求 | 说明 |
|------|------|
| 最小化安装 | 不安装不需要的软件包 |
| 服务最小权限 | 各服务以独立用户运行，仅必要 capabilities |
| 防火墙默认拒绝 | input/forward 默认 drop，仅放行必要端口 |
| 审计日志 | 所有配置变更记录审计日志 |
| SSH 加固 | 禁止 root 密码登录，仅允许密钥认证 |

### 3.3 通信安全

| 需求 | 说明 |
|------|------|
| API 通信加密 | HTTPS (TLS 1.2+) |
| 配置文件权限 | 0600，属主 ubunturouter |
| 敏感信息保护 | 密码/密钥不以明文存储在配置文件中（可选加密） |

## 8. 万兆场景特殊需求

万兆性能目标对系统架构有额外要求：

| 需求 | 说明 |
|------|------|
| RPS/RFS | 多核网卡中断负载均衡，单核无法线速万兆 |
| IRQ Affinity | 网卡中断绑定到指定 CPU 核心 |
| XDP 快速路径 | 可选 XDP redirect 绕过内核协议栈 |
| conntrack 调优 | hashsize/max 根据内存自动计算 |
| TCP 缓冲区调优 | rmem/wmem 针对万兆优化 |
| nftables 集合 | 使用 named set 替代逐条规则，O(1) 查找 |
| NUMA 感知 | 多路服务器需确保网卡和 CPU 同 NUMA 节点 |
| offload 支持 | 开启网卡 LRO/GRO/GSO 硬件卸载 |

## 9. ARM 平台特殊需求

| 需求 | 说明 |
|------|------|
| 内核适配 | 部分ARM平台需主线内核外的驱动补丁 |
| 性能调优 | ARM 缓存/TLB 特性不同于 x86，需针对性调优 |
| 散热管理 | ARM 开发板散热有限，需监控温度并降频保护 |
| 存储适配 | eMMC/SD 卡性能差异大，减少写操作 |
| 网卡驱动 | ARM 内置网卡驱动可能需要 DKMS 编译 |
