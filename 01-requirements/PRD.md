# 产品需求文档 (PRD)

> 版本: v1.0 | 日期: 2026-04-25 | 状态: 评审中

## 1. 产品定位

**UbuntuRouter** 是一个基于 Ubuntu Linux 的现代软路由管理框架，面向具备技术背景的个人家庭用户和 SOHO 小型办公场景，提供开箱即用的路由器体验，同时保留 Ubuntu 生态的全部扩展能力。

**一句话描述**：让 Ubuntu 成为比 OpenWrt 更强大、比 pfSense 更灵活的软路由操作系统。

## 2. 目标用户

### 2.1 主要用户群：技术型家庭用户

| 属性 | 描述 |
|------|------|
| 技术水平 | 具备 Linux 基础，能操作命令行 |
| 典型硬件 | x86 小主机 (N100/J4125)、ARM 开发板 (RK3568/RPi5)、虚拟机 |
| 核心诉求 | 千兆/万兆 NAT 转发、广告过滤、VPN 远程回家、多 WAN、VLAN 隔离 |
| 痛点 | OpenWrt x86 扩展性差，pfSense 闭源且贵，手配 Ubuntu 太复杂 |

### 2.2 次要用户群：SOHO 小型办公

| 属性 | 描述 |
|------|------|
| 技术水平 | 有 IT 兼职人员或外包运维 |
| 典型硬件 | x86 工控机、1U 服务器、云虚拟机 |
| 核心诉求 | 多 WAN 负载均衡、VPN 站点互联、VLAN 办公隔离、稳定可靠 |
| 痛点 | 商业路由器贵，OpenWrt 企业功能弱，VyOS 无 Web GUI |

**网口自动分配规则**：

```
启动时自动检测所有物理以太网口
    │
    ├─ 检测到 1 个网口
    │    → 单网口模式 (WANLAN)
    │    → 该网口上创建 VLAN 子接口：
    │       - wan0 (原始接口, DHCP 获取上游 IP)
    │       - lan0 (br-lan, 192.168.21.1/24)
    │    → 或：单网口 NAT 模式，该口作为 LAN，
    │       上行通过 PPPoE/DHCP 获取 WAN IP
    │
    ├─ 检测到 2 个网口
    │    → 速率较低的网口 = WAN (enp1s0)
    │    → 速率较高的网口 = LAN (enp2s0)
    │
    └─ 检测到 3+ 个网口
         → 速率最低的网口 = WAN
         → 其余网口桥接 = br-lan
         → WAN: DHCP (默认)
         → LAN: 192.168.21.1/24
         → DHCP: 192.168.21.50-200
```

## 5. 硬件平台支持

| 平台 | 优先级 | 典型硬件 | 说明 |
|------|--------|----------|------|
| x86_64 | P0 | N100/J4125 小主机、工控机 | 主力平台，支持KVM虚拟化 |
| ARM64 (aarch64) | P0 | RK3568/RK3588、树莓派5、NanoPi | KVM虚拟化需硬件支持(部分SoC不支持) |
| 虚拟机 (KVM/Proxmox) | P0 | x86/ARM 虚拟机 | VFIO 直通网口，嵌套虚拟化可选 |
| 云服务器 | P1 | 阿里云/AWS VPC 路由 | 仅软件转发，无KVM虚拟化 |
| ARM32 (armhf) | P2 | 树莓派4 (32bit) | 性能受限，不支持KVM |

**虚拟化硬件要求**：
- KVM 虚拟机需 CPU 支持 VT-x (x86) / VHE (ARM)
- VFIO 直通需 IOMMU 支持 (VT-d / ARM SMMU)
- 建议最低 8GB RAM（同时运行路由+VM+容器）
- 建议最低 64GB 存储（VM 镜像占用较大）
