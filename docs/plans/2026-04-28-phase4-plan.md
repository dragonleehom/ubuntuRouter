> **执行方式**: 使用 subagent-driven-development，每项任务独立开发

## 4.1 主题/样式选择 — 前端 1d

**目标**: 提供明暗主题切换功能，允许用户在暗色(当前)和亮色(毛玻璃)主题间切换

**涉及文件**:
- `web/src/main.js` — 导入主题切换逻辑
- 新建 `web/src/stores/theme.js` — Pinia store 管理主题状态
- 新建 `web/src/assets/theme-dark.css` — 从 `App.vue` 提取当前暗色样式
- `web/src/assets/theme-light-glass.css` — 已有亮色主题（已存在）
- 修改 `web/src/layouts/MainLayout.vue` — 顶部栏添加主题切换按钮
- `web/src/App.vue` — 移除内联暗色样式，改为动态切换

**实现方案**:
- 使用 Pinia store (`theme.js`) 存储当前主题名，持久化到 `localStorage`
- 在 `document.documentElement` 上切换 `data-theme` 属性
- 每个组件使用 CSS 变量 + `data-theme` 选择器适配
- 顶部栏右侧添加太阳/月亮图标切换按钮

## 4.2 应用评分体系 — 后端 1d + 前端 0.5d

**目标**: 允许用户对已安装的应用进行评分（1-5星），显示平均评分

**涉及文件**:
- 新建 `src/ubunturouter/api/routes/ratings.py`
- 修改 `src/ubunturouter/api/main.py` — 注册路由
- 修改 `web/src/views/appstore/AppStore.vue` — 评分展示
- 新建 `web/src/views/appstore/RatingDialog.vue` — 评分弹窗

**后端 API**:
```
GET    /api/v1/ratings/{app_id}              # 获取应用评分
POST   /api/v1/ratings/{app_id}              # 提交评分（1-5）
GET    /api/v1/ratings/{app_id}/my           # 获取当前用户评分
```

**数据存储**: JSON 文件 `/opt/ubunturouter/data/ratings.json`

**前端**:
- 应用详情弹窗中显示平均评分和星数
- 已安装用户可点击评分
- 评分后显示

## 4.3 Docker 事件日志 — 后端 0.5d

**目标**: 提供 Docker 事件日志查看端点

**涉及文件**:
- 修改 `src/ubunturouter/api/routes/containers.py` — 新增端点

**后端 API**:
```
GET /api/v1/containers/events?since=1h&until=now
```

**实现**:
- 执行 `docker events --since 1h --format '{{json .}}'` 获取事件
- 支持 `since` 参数（1h, 5m, 24h 等格式）
- 返回事件列表，包含 type, action, id, from, time, status

## 4.4 语言切换 — 前端 1d

**目标**: 支持中英文界面切换

**涉及文件**:
- 新建 `web/src/stores/locale.js` — Pinia store 管理语言状态
- 新建 `web/src/locales/zh-CN.js` — 中文翻译
- 新建 `web/src/locales/en.js` — 英文翻译
- 修改 `web/src/main.js` — 注册 i18n
- 修改 `web/src/layouts/MainLayout.vue` — 顶部栏添加语言切换

**实现方案**:
- 使用 Vue 3 响应式对象做轻量 i18n，不引入 vue-i18n 依赖
- 定义翻译对象：`{ "应用市场": "App Store", "系统设置": "System Settings" }`
- 提供 `$t()` 全局方法或 composable
- 持久化到 `localStorage`
