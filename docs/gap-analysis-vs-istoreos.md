# UbuntuRouter vs iStoreOS — 差距分析报告

> 分析日期: 2026-04-27
> 方法: 实际登录两个系统的管理界面逐页对比
> 基准版本: iStoreOS 24.10.2 / UbuntuRouter 最新代码

---

## 一、综述对比

| 维度 | UbuntuRouter | iStoreOS | 差距等级 |
|------|-------------|----------|---------|
| 功能模块数量 | 24 个导航项 | 7 大模块 + LuCI 底层 | 数量接近 |

## 二、逐模块差距分析

### 2.1 仪表盘（路由状态）★★★★★

#### iStoreOS 有而 UbuntuRouter 没有的

| 功能 | 说明 | 重要性 |
|------|------|--------|
| **实时上下行流量图** | Canvas 动画曲线，流量走势可视化 | ★★★★★ |
| **当前流量数值** | Upload/Download 实时速率 | ★★★★★ |
| **快捷操作面板** | 8 个一键按钮: Network Guide / Terminal / OTA / LAN Settings / DNS Settings / Feeds Mirror / Enable Sandbox | ★★★★ |
| **联网状态总览** | 联网状态、连接方式、在线设备数一栏展示 | ★★★★ |
| **磁盘信息 + 文件管理器入口** | 磁盘列表 + 一键进入文件管理 | ★★★ |
| **共享状态面板** | SAMBA/WEBDAV/LinkEase 当前状态 | ★★★ |
| **Docker 状态面板** | Docker 运行状态 + 根目录 | ★★★ |
| **下载服务状态** | Aria2/qBittorrent/Transmission 状态 | ★★★ |
| **DDNS 状态预览** | DDNSTO / myddns_ipv4 / myddns_ipv6 状态 | ★★★ |
| **系统信息卡片** | CPU 温度/型号/固件版本/内核/运行时间/系统时间 | ★★ |

#### UbuntuRouter 当前仪表盘

- 系统资源（CPU/内存/磁盘/运行时间）⭐ 有
- 网络拓扑图（ECharts 力导向图）⭐ 有
- 接口状态列表 ⭐ 有
- 服务状态列表（启动/停止按钮）⭐ 有

**差距总结**: 仪表盘是用户第一眼看到的东西，iStoreOS 的仪表盘是一个**集成控制中心**，UbuntuRouter 的仪表盘只是一个基本的状态展示页。最致命的是缺少实时流量和快捷操作。

### 2.2 网络配置 ★★★★★

#### iStoreOS 的连接配置方式

iStoreOS 提供完整的网络配置向导，用户体验极佳：

| 模块 | 功能 | 用户操作方式 |
|------|------|-------------|
| **拨号连接** | PPPoE 宽带拨号 | 填写账号密码 → 保存生效 |
| **DHCP 连接** | 自动获取 IP | 点击选择 → 保存生效 |
| **旁路由连接** | 单臂路由模式 | 填 IP/掩码/网关/DNS → 选 DHCP 开关 → 保存 |
| **内网配置** | LAN 口 IP/子网 | 填 IP/子网掩码 → 保持或修改 DHCP → 保存 |
| **DNS 配置** | DNS 服务器 | 下拉选择"运营商DNS"或"自定义DNS" → 保存 |

所有配置都是 **表单填写 → 保存 → 立即生效**，零门槛。

#### UbuntuRouter 当前网络配置

| 功能 | 状态 | 说明 |
|------|------|------|
| 网络接口列表 | ⭐ 有 | 只读查看 + 编辑 IP 按钮 |
| PPPoE 拨号 | ⭐ 有 | 独立页面，但不如 iStoreOS 直观 |
| 网络配置向导 | ❌ 无 | 需要手动修改 YAML 配置 |
| 旁路由模式 | ❌ 无 | 不支持 |
| 内网配置 | ❌ 无 | 不支持 |
| DNS 配置 | 分散在 DHCP/DNS 页 | 没有一键切换运营商DNS/自定义DNS |
| WAN 连接类型切换 | ❌ 无 | 没有拨号/DHCP/静态 三选一 |

**差距总结**: iStoreOS 网络配置的核心理念是**"你是什么上网方式？选一个→填几项→保存"**。UbuntuRouter 需要用户理解 YAML 结构或者找分散的页面去配置。这是对新用户最不友好的地方。

### 2.6 LuCI 底层能力 ★★★★★

iStoreOS 基于 OpenWrt，可以随时进入**完整 LuCI 高级模式**，获得全套专业网络管理工具：

| 菜单 | 功能 |
|------|------|
| Status | 路由表、系统日志、内核日志、防火墙、进程、实时图表、带宽监控 |
| System | 主机名/时区/密码/SSH/软件包/启动项/计划任务/重启/备份 |
| Docker | 完整 Docker 容器管理 |
| Services | 所有运行服务管理 |
| NAS | 完整 NAS 配置 |
| Network | 接口/DHCP/Wireless/防火墙/路由/诊断 |

UbuntuRouter 没有这个底层安全网——如果用户需要配置标准网络功能之外的东西，就无处可去了。（当前只有 Web 终端 ttyd）

## 四、核心差距根因分析

### 根因 1: 技术路线差异

```
iStoreOS:
  OpenWrt (内核层) → LuCI (管理面板) → iStoreOS 主题美化

UbuntuRouter:
  Ubuntu Server → 自建 Python/FastAPI → Vue3 前端
```

iStoreOS 的 OpenWrt 底座提供了 **20 年积累的网络配置能力**。UbuntuRouter 从零自建，所有网络配置都需要自己写代码实现。

### 根因 2: iStoreOS 的"生态加成"

iStoreOS 的仪表盘快捷面板、共享管理、DDNS 状态、下载服务状态——这些都是通过**应用商店安装的应用提供的**。iStoreOS 的应用知道如何把自己的状态注册到仪表盘。UbuntuRouter 的应用市场是纯 Docker 仓库，应用自身不会向系统注册任何状态。

### 根因 3: iStoreOS 用户基数大、迭代多

iStoreOS 社区数十万用户，应用评分/下载量/分类都是社区长期积累的结果。UbuntuRouter 目前只有一个人开发。
